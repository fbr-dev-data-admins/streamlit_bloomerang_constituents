# Bloomerang Constituent Export

A Streamlit application for exporting and transforming constituent data from the Bloomerang CRM.

## Prerequisites

- Python 3.10+
- A Bloomerang v2 private API key (must be generated by a Bloomerang Administrator user)

## Installation

1. Clone or download this repository

2. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```

3. Copy the example environment file and add your API key:
   ```bash
   cp .env.example .env
   ```
   
   Edit `.env` and replace `your_private_api_key_here` with your actual Bloomerang private API key.

4. (Optional) Edit `config/config.yaml` to configure:
   - **excluded_groups**: Group names to filter out from export
   - **supporter_type_mapping**: Mappings for Supporter Type custom field values
   - **name_prefixes/suffixes**: Titles and suffixes for name parsing
   - **name_exception_keywords**: Keywords that trigger manual review for individual names

## Running the Application

```bash
streamlit run app.py
```

The application will open in your default web browser.

## Usage

1. Select a date range using the sidebar controls
2. Click "Run Export" to fetch and transform records
3. Review the summary statistics and data previews
4. Download the generated Excel files

## Output Files

The application generates up to three Excel files:

### `{date_range}_individuals.xlsx`
Contains transformed Individual constituent records with:
- Parsed name fields (First Name, Last Name, Title, Suffix)
- Spouse information for couple records
- Contact information (address, email, phone)
- Addressee/Salutation codes

### `{date_range}_organizations.xlsx`
Contains transformed Organization constituent records with:
- Organization name
- Primary contact information
- Contact information (address, email, phone)
- Supporter Type

### `{date_range}_manual_review.xlsx`
**Only created when Individual records have names that cannot be automatically parsed.**

This file contains:
- All raw fields from the API response
- The reason for requiring manual review (always "Unparseable Name")

Records appear here when:
- The name is empty
- The name contains business-related keywords (Family, Foundation, etc.)
- The name is entirely uppercase or lowercase
- The name cannot be parsed into first/last name structure

## Configuration

### Excluded Groups

To exclude constituents who belong to certain groups, edit `config/config.yaml`:

```yaml
excluded_groups:
  - "Facebook Leads"
  - "Another Group Name"
```

Groups are matched case-insensitively against the `GroupsDetails` array on each constituent record.

### Supporter Type Mapping

To map Bloomerang Supporter Type values to output values:

```yaml
supporter_type_mapping:
  "Individual": "Individual"
  "Organization": "Organization"
  "Major Donor": "Major Donor"
```

Values not in the mapping are passed through as-is.

## API Key Notes

- The API key must be a v2 private key
- It must be generated by a user with Administrator privileges in Bloomerang
- The key is used with HTTP Basic Authentication (key as username, blank password)

## Troubleshooting

- **"BLOOMERANG_API_KEY environment variable is not set"**: Ensure you've created the `.env` file with your API key
- **Rate limiting errors**: The application automatically retries with backoff, but very large exports may hit limits
- **Parse errors**: Records with unparseable names go to the manual review file; check there for issues
